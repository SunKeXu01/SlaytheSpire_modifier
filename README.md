# 杀戮尖塔助手

一个适用于 Windows 64 位 Steam 版《杀戮尖塔》的单机辅助工具。游戏启动后运行助手，即可锁定当前可用能量、设置金币或恢复角色生命值。

> 本工具仅用于单机游戏。请勿用于任何带有竞技、公平排名或多人对战性质的场景。

## 功能

- 自动识别已经启动的《杀戮尖塔》进程
- 将当前可用能量持续锁定为设定值（跨回合自动维持）
- 将金币直接设置为指定值
- 恢复指定数量的生命值
- 一键恢复全部生命值
- 可在界面中调整能量、金币和恢复生命的数值
- 自动检查游戏内部结构是否兼容
- 防止助手重复启动
- 记录连接和错误日志，方便排查问题

## 使用方法

1. 启动 Steam 版《杀戮尖塔》。
2. 进入一局游戏，最好停留在战斗中轮到玩家出牌的画面。
3. 双击运行 `杀戮尖塔助手 v1.0.2.exe`。
4. 等待界面显示“已连接《杀戮尖塔》”。
5. 点击界面按钮或使用快捷键。

快捷键：

| 快捷键 | 功能 |
| --- | --- |
| `F1` | 开启或关闭能量锁定 |
| `F2` | 按界面设定值恢复生命 |
| `F3` | 恢复全部生命 |
| `F4` | 将金币设置为界面中的数值 |

## 能量数值说明

游戏能量会显示为类似 `0/17` 的形式：

- 斜杠左侧是本回合当前可用能量
- 斜杠右侧是本回合能量总量

“锁定能量”只持续修改斜杠左侧的当前可用能量，不会改变右侧总量。锁定启用后，出牌消耗或进入下一回合都会自动恢复为设定值；再次点击按钮、按 F1 或退出助手即可关闭锁定。

## 运行环境

- Windows 10 或 Windows 11 64 位
- Steam 版《杀戮尖塔》64 位
- 游戏与助手需要使用相同权限运行
- 系统需要支持 .NET Framework 4.x；Windows 10/11 通常已经自带

当前已验证的游戏文件指纹：

```text
desktop-1.0.jar
SHA-256: CFAD868AC8D65A88E71A0BF096FB09F78811E553EFFE0787C5309A655E081673
```

其他版本若游戏内部字段保持一致，也可能正常使用。游戏更新后如果助手提示版本不兼容，需要重新适配。

## 常见问题

### 助手显示“尚未检测到游戏”

确认游戏窗口已经完全启动。助手支持识别窗口标题为 `Slay the Spire` 的 `javaw.exe` 游戏进程。

### 助手显示“连接失败”

首先尝试让游戏和助手都以普通权限运行。如果游戏以管理员身份启动，助手也必须以管理员身份启动。

### 安全软件产生提示

助手需要连接游戏进程并读取游戏内置 Java 运行时中的角色数据，这类行为可能触发安全软件的启发式检测。建议只从本仓库的 Releases 页面下载，并使用发布页面提供的 SHA-256 校验值核对文件。

### 点击锁定能量但没有生效

请确认已经进入战斗。主菜单、地图或尚未开始一局时，助手不会修改能量。

### 日志在哪里

日志文件位于：

```text
%LOCALAPPDATA%\SlayTheSpireAssistant\assistant.log
```

反馈问题时请附上日志内容、游戏版本以及问题截图。

## 从源码构建

构建环境：

- Windows 64 位
- Visual Studio C++ Build Tools（支持 x64）
- JDK 8（需要 `jni.h`）
- .NET Framework 4.x C# 编译器

源码结构：

```text
src/AssistantBridge.cpp          游戏运行时连接组件
src/SlayTheSpireAssistant.cs     Windows 界面、进程识别与快捷键
```

连接组件被作为资源嵌入最终 EXE，运行时会释放到系统临时目录，所以分发时只需要发送一个 EXE。
